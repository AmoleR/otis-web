Notes for OTIS
  • Problem articles should be titled by PUID. Use the header [problem PUID][/problem] (with a new line between).
  • Using [statement] generates the problem statement.
  • Diamond articles should be titled by the achievement name. Use the header [code CODE][/code].
  • Unit articles should be titled by the short name shown on the catalog and on the OTIS-WEB portal. Use the header [unit SLUG][/unit]. (The slug is the natural part of the unit's filename, e.g. "equality".)
  • There is also a pair [generic][/generic] that makes an empty table.
  • You can add pairs using the syntax Key | Value within any of the four blocks above.
  • Articles generally follow title case.
